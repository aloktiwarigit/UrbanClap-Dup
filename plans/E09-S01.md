# E09-S01 — Owner Live Ops Dashboard

**Epic:** E09 — Owner Admin Experience

**Tier:** Feature
**UX demo:** `docs/ux-demos/owner-live-ops-dashboard.html` — "Operations Observatory"
**Blast radius:** API (3 new endpoints + 3 schemas + OpenAPI) + admin-web (1 page + 4 components)
**Wall-clock target:** 1.5–2.5h

---

## Real-time approach (architectural note)

Per ADR-0002, the full vision is Cosmos change-feed → Azure Function trigger → FCM topic → admin web. For MVP this story uses **30-second client-side polling** against the REST endpoints. This is sufficient for the owner dashboard at pilot scale and eliminates Firebase JS SDK complexity. FCM-based streaming is a Phase 2 enhancement; document it as a `// TODO(E09-S01-v2): replace with FCM topic listener` comment at call sites.

## Map approach

`@vis.gl/react-google-maps` is **not** currently in `admin-web/package.json`. The UX demo itself uses a **purely CSS-drawn map** (stylised grid lines + absolutely-positioned pins, no real mapping SDK). We replicate this exactly — no external map SDK, no paid SaaS. Tech pin positions are computed as normalised percentages derived from lat/lng relative to a bounding box. This keeps the dep count flat and matches the demo pixel-for-pixel.

### Endpoint specifications

#### `GET /v1/admin/dashboard/summary`

- Auth: `requireAdmin(['super-admin', 'ops-manager'])`
- Query Cosmos `bookings` container for today's date range (`createdAt >= startOfDay`)
  - `bookingsToday`: COUNT of bookings where `createdAt` is today
  - `gmvToday`: SUM of `amount` for `status = 'completed'` today
  - `commissionToday`: `gmvToday * commissionRate` (env var `COMMISSION_RATE`, default 0.225)
  - `complaintsOpen`: COUNT from `complaints` container where `status = 'open'`
  - `techsOnDuty`: COUNT from `technicians` container where `isOnDuty = true`
  - `payoutsPending`: SUM of `pendingPayout` from `wallet_ledger` container
- Response: `DashboardSummaryResponseSchema`
- Cache-Control: `no-store` (live data)
- On Cosmos error: 502 with `{ code: 'UPSTREAM_ERROR' }`

#### `GET /v1/admin/dashboard/feed`

- Auth: `requireAdmin(['super-admin', 'ops-manager'])`
- Query `booking_events` container: newest-first, limit 50
  - Partition: scan across all partitions (admin endpoint, acceptable RU cost at pilot scale)
  - Order by `createdAt DESC`
- Response: `BookingEventsResponseSchema`
- Cache-Control: `no-store`

#### `GET /v1/admin/dashboard/tech-locations`

- Auth: `requireAdmin(['super-admin', 'ops-manager'])`
- Query `technicians` container where `isOnDuty = true`
  - Project: `technicianId, name, serviceType, lat, lng, state, updatedAt`
- Response: `TechLocationsResponseSchema`
- Cache-Control: `max-age=30` (matches client poll interval)

### TDD contract per test file

Each test file must cover:
1. Happy path — returns correctly shaped response (validated with `safeParse`)
2. Auth guard — unauthenticated request returns 401
3. Cosmos error — mocked Cosmos throws → handler returns 502 `UPSTREAM_ERROR`
4. Schema validation — response shape matches Zod schema

Use `vitest` + mock `CosmosClient` (import mock via `vi.mock`). Do **not** use Testcontainers for this story (no integration tests needed for read-only queries; those belong in E09-S02 acceptance testing).

### Acceptance criteria

- All 3 test files green (`pnpm --filter api test`)
- No `any` types in handler files
- `requireAdmin` wraps every handler (copy pattern from `api/src/functions/admin/me.ts`)
- `app.http()` route registrations follow the `v1/admin/dashboard/*` pattern

#### `admin-web/src/components/dashboard/CounterStrip.tsx`
- Server-compatible (no hooks, receives props)

- Tile accents: `data-accent` attr drives the top-edge gradient line colour
- 6 tiles:

  2. **GMV · Today** — `₹{gmvToday/100}` (accent: teal)
  3. **Commission** — `₹{commissionToday/100}` + blended rate (accent: ember)
  4. **Payouts · Pending** — `₹{payoutsPending/100}` (accent: ember)
  5. **Complaints · Open** — `complaintsOpen` (accent: coral)
  6. **Techs · On duty** — `techsOnDuty` (accent: teal)
- Count-up animation: `requestAnimationFrame` eased cubic, 1400ms, triggers on mount via `useEffect`
