## Brainstorm (embedded, feature tier)

**Key questions resolved before coding:**

1. **Which JSON token format?** W3C DTCG (`$value` / `$type`). Reasons: (a) natively imported by Token Studio free plugin, (b) Style Dictionary can consume it in Phase 2, (c) it's the emerging standard (Figma's own variables JSON export converges to this). Rejected: proprietary Theo format (no Figma native import), Style Dictionary v3 JSON (older, more verbose).

2. **How does drift detection work?** Python script reads `figma/variables.json` and regex-parses the Kotlin token files. Compares hex values for color, dp integers for dimension tokens. Exits non-zero on first mismatch. Runs in CI `design-system-ship.yml` so every PR touching `design-system/` or `figma/` is checked.

3. **What's the figma-code-connect story on free tier?** The CLI tool (`@figma/code-connect`) is OSS; creating `.figma.kts` files is free. The `figma connect publish` step calls the Figma REST API and requires Dev Mode (paid). Decision: create the `.figma.kts` stubs in `figma/code-connect/` as documentation artifacts; the README documents when/how to activate them. Zero paid cost today, zero throwaway work when team upgrades.

4. **Where is source of truth?** Kotlin (Color.kt, Spacing.kt, etc.) — unchanged from ADR-0010. `figma/variables.json` is a derived mirror. The CI drift check enforces this direction.

5. **Do Paparazzi goldens need updating?** No — we're not touching any Composable source; tokens are unchanged. CI will confirm.

**Files:** `figma/variables.json`, `figma/README.md`, `figma/code-connect/HomeservicesTheme.figma.kts`

`figma/variables.json` — W3C DTCG nested structure:
- `color/brand` — primary (light+dark), primaryHover (light+dark), accent (light+dark)
- `color/semantic` — success, warning, danger, info (light+dark each)
- `color/neutral` — 0, 50, 100, 200, 500, 900 (light+dark each)
- `color/extended` — verified, neighbourhood (light+dark)
- `typography` — 10 slots: display.xl/lg, title.lg/md/sm, body.lg/md/sm, label.lg/sm
- `spacing` — space.0..space.24 (10 steps)
- `motion/duration` — fast, base, medium, slow
- `motion/easing` — standard, emphasizedDecelerate
- `elevation` — elev.0..elev.4 (Dp values)
- `radius` — sm, md, lg, xl, full

`figma/README.md` — manual import flow using Token Studio plugin (free).

`figma/code-connect/HomeservicesTheme.figma.kts` — stub with `figma("URL", "ComponentName") {}` blocks for HomeservicesTheme composable; comment explains Dev Mode requirement.
